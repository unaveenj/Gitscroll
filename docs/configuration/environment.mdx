GitScroll reads environment variables at server startup. Client components never have access to these values.

## Variables

<ParamField path="GITHUB_TOKEN" type="string" required>
  A GitHub personal access token used to authenticate requests to the GitHub Search API.

  Without this variable the app falls back to 12 built-in mock repositories. The feed pipeline (categorisation, interleaving, pagination) runs identically either way.

  See [GitHub token setup](/configuration/github-token) for instructions.
</ParamField>

## Where to set them

### Local development

Create a `.env.local` file in the project root (this file is git-ignored):

```bash
GITHUB_TOKEN=ghp_xxxxxxxxxxxxxxxxxxxx
```

Next.js loads `.env.local` automatically. You do not need to prefix the variable with `NEXT_PUBLIC_` — it is only used on the server.

### Vercel

In your Vercel project dashboard go to **Settings → Environment Variables** and add `GITHUB_TOKEN` with your token value. Set the environment to **Production**, **Preview**, and **Development** as appropriate.

### Other platforms

Set `GITHUB_TOKEN` as a standard environment variable through your platform's secret management UI or CLI. The variable is read via `process.env.GITHUB_TOKEN` in `src/services/githubService.ts`.

## Security

<Warning>
Never set `NEXT_PUBLIC_GITHUB_TOKEN`. The `NEXT_PUBLIC_` prefix exposes variables to the browser bundle. Your token would be visible to anyone who inspects the page source.
</Warning>

`githubService.ts` is annotated as server-only and imported exclusively by `src/app/api/repos/route.ts`. Next.js will throw a build error if it is ever imported from a client component.
