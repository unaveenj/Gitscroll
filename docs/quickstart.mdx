## Prerequisites

- Node.js 18 or later
- A GitHub Personal Access Token (optional but recommended — without one the app uses mock data)

## Setup

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/your-username/gitscroll.git
    cd gitscroll
    ```
  </Step>

  <Step title="Install dependencies">
    <Tabs>
      <Tab title="npm">
        ```bash
        npm install
        ```
      </Tab>
      <Tab title="yarn">
        ```bash
        yarn install
        ```
      </Tab>
      <Tab title="pnpm">
        ```bash
        pnpm install
        ```
      </Tab>
    </Tabs>
  </Step>

  <Step title="Add your GitHub token">
    Create a `.env.local` file in the project root:

    ```bash
    GITHUB_TOKEN=ghp_your_token_here
    ```

    <Note>
      If you skip this step, GitScroll falls back to 12 built-in mock repositories so you can still explore the UI.
    </Note>

    See [GitHub token setup](/configuration/github-token) for instructions on creating a token with the right scopes.
  </Step>

  <Step title="Start the dev server">
    ```bash
    npm run dev
    ```

    Open [http://localhost:3000](http://localhost:3000) in your browser.
  </Step>
</Steps>

## What you should see

On first load, GitScroll fetches a random page of repositories from GitHub and renders a stack of flash cards. Swipe a card upward (or drag it past the threshold) to advance to the next one.

| Action | Result |
|--------|--------|
| Drag card up past threshold | Dismiss and advance to next card |
| Tap or click card | Flip to the detail view |
| Press Save | Persist repo to local favourites |
| Press Open on GitHub | Open the repository in a new tab |
| Press Space or ↑ | Keyboard advance to next card |
| Press 🎲 Surprise Me | Jump to a random card in the deck |

## Running without a GitHub token

When `GITHUB_TOKEN` is not set, the server logs a warning and returns 12 pre-defined mock repositories:

```
[githubService] GITHUB_TOKEN not set — using mock data
```

The full UI — including swipe, flip, favourites, and theme toggle — works normally with mock data.

## Building for production

```bash
npm run build
npm start
```

The build output is a standard Next.js production bundle.
